## 2. WFL SYNTAX SPECIFICATION

### Core Grammar Rules

```bnf
Expression    ::= Component ("+" Component)*
Component     ::= Type "." Modifier* ("{" Children "}")? (":" Content)?
Type          ::= [a-z]{2,3}                    ; 2-3 letter codes
Modifier      ::= [a-z]{1,3} | "#" [a-z0-9]+   ; properties/variants
Children      ::= Expression | "[" Expression ("," Expression)* "]"
Content       ::= "\"" String "\""
Variable      ::= "@" Identifier
Logic         ::= "!" Action "(" Args ")"
```

### Registry Structure

```typescript
interface ComponentRegistry {
  [token: string]: {
    // Metadata
    category: 'layout' | 'ui' | 'data' | 'logic';
    source: 'shadcn' | 'mui' | 'custom' | 'radix';
    version: string;
    
    // Code mapping
    react: {
      import: string;
      component: string;
      propsMap: Record<string, string>;
    };
    
    // Style mapping
    styles: {
      base: string;
      variants: Record<string, string>;
      modifiers: Record<string, string>;

    // Token mappings (2-3 letter codes)
    tokens: {
      type: string;        // e.g., "btn" = Button

        lg: 'size-large',  // size
        gls: 'glass',      // effect
        r: 'right',        // alignment
        c: 'center',
        l: 'left'
      }
    }
  }
}
```

## 4. TRANSPILER LOGIC (Deep Dive)

### Stage 1: Tokenizer (Lexer)

```javascript
class WFLTokenizer {
  tokenize(input) {
    const tokens = [];
    const patterns = [
      { type: 'TYPE', regex: /^[a-z]{2,3}/ },           // btn, nav, stk
      { type: 'DOT', regex: /^\./ },                    // .
      { type: 'MOD', regex: /^[a-z0-9#]+/ },            // pri, lg, #ff5733
      { type: 'LBRACE', regex: /^\{/ },                 // {
      { type: 'RBRACE', regex: /^\}/ },                 // }
      { type: 'PLUS', regex: /^\+/ },                   // +
      { type: 'COLON', regex: /^:/ },                   // :
      { type: 'STRING', regex: /^"[^"]*"/ },            // "text"
      { type: 'AT', regex: /^@/ },                      // @
      { type: 'BANG', regex: /^!/ },                    // !
      { type: 'LPAREN', regex: /^\(/ },                 // (
      { type: 'RPAREN', regex: /^\)/ },                 // )
      { type: 'IDENT', regex: /^[a-zA-Z]+/ }            // identifiers
    ];
    
    // Implementation parses left-to-right
    // btn.pri.lg → [TYPE:btn, DOT, MOD:pri, DOT, MOD:lg]
  }
}
```

### Stage 2: Parser (AST Generation)

```javascript
class WFLParser {
  parse(tokens) {
    // Builds Abstract Syntax Tree
    // nav.dk { btn.pri } becomes:
    /*
    {
      type: "Component",
      element: "nav",
      modifiers: ["dk"],
      children: [
        {
          type: "Component", 
          element: "btn",
          modifiers: ["pri"],
          children: []
        }
      ]
    }
    */
  }
}
```

### Stage 3: Resolver (Component Lookup)

```javascript
class WFLResolver {
  resolve(astNode) {
    const { element, modifiers } = astNode;
    
    // 1. Lookup base component
    const componentDef = registry[element];
    if (!componentDef) throw new Error(`Unknown token: ${element}`);
    
    // 2. Resolve modifiers to props
    const props = {};
    const classes = [];
    
    modifiers.forEach(mod => {
      if (componentDef.react.propsMap[mod]) {
        // It's a prop variant
        const [key, value] = componentDef.react.propsMap[mod].split('=');
        props[key] = value;
      }
      if (componentDef.styles?.variants?.[mod]) {
        // It's a style variant
        classes.push(componentDef.styles.variants[mod]);
      }
    });
    
    return {
      component: componentDef.react.component,
      import: componentDef.react.import,
      props,
      classes,
      children: astNode.children.map(c => this.resolve(c))
    };
  }
}
```

### Stage 4: Code Generator

```javascript
class WFLGenerator {
  generate(resolvedTree, target = 'nextjs') {
    const imports = new Set();
    const components = [];
    
    this.traverse(resolvedTree, (node) => {
      // Collect imports
      imports.add(node.import);
      
      // Generate JSX
      const propsString = Object.entries(node.props)
        .map(([k, v]) => `${k}=${v}`)
        .join(' ');
      
      const classString = node.classes.join(' ');
      
      components.push(
        `<${node.component} ${propsString} className="${classString}">
          ${node.children.map(c => this.generate(c)).join('')}
        </${node.component}>`
      );
    });
    
    return {
      imports: Array.from(imports),
      jsx: components.join('\n'),
      css: this.extractStyles(components)
    };
  }
}
```

## 6. COMPILATION PIPELINE

```
WFL Source
    │
    ▼
[Tokenizer] ──▶ Token Stream
    │
    ▼
[Parser] ────▶ Abstract Syntax Tree
    │
    ▼
[Resolver] ──▶ Component Tree + Props
    │
    ▼
[Optimizer] ─▶ (Deduplicate imports, Merge classes)
    │
    ▼
[Generator] ─▶ Target Output
    │
    ├──▶ Next.js / React
    ├──▶ Vue
    ├──▶ Svelte
    └──▶ Raw HTML/CSS
```
